# 张麓：第一步能力资产诊断最终交付版

> 用途：作为张麓后续复盘能力资产、搭建知识体系、进入商业切入口选择的基础材料。

> 产品阶段说明：这是“能力资产与商业切入口诊断”产品的第一步。第一步先不急着判断应该卖什么、服务谁、做哪个商业方向，而是先把真实拥有的能力资产、知识体系资产和当前负债挖出来、校准清楚。完成这一步之后，第二步再进入“商业切入口选择和收敛”。

## 0. 这份交付怎么用

这是本次“能力资产与商业切入口诊断”第一步的最终交付版本。

它把会前材料、近期 AI 对话、历史职场经历和真人访谈校准结果合在一起，形成四部分：

1. 能力资产地图
2. 知识体系资产分层
3. 能力资产负债表
4. 第二步候选能力池

### 0.1 我们现在拥有了哪些证据

| 证据层 | 作用 | 材料 |
|---|---|---|
| 主证据：近期行动与沉淀 | 发现当前真实能力 | AI 对话、Obsidian 人机协作目录、价值萃取框架、JTBD 拆解、OSCAR 调研 |
| 旁证：历史职场经历 | 验证能力是否长期存在 | 数字孪生智能协商、AutoBA、智能情景感知、移动支付、金融信息推荐 |
| 补充线索：受阻与不确定性 | 后续负债判断时参考 | 商业化闭环不足、项目规模化不足、平台资源依赖等线索 |
| 校准证据：真人访谈 | 修正能力命名、边界、等级和负债优先级 | 2026 年 6 月 23 日一对一访谈，以及《张麓-能力挖掘校准与调整记录》 |

使用原则：

- 近期材料负责发现能力。
- 历史材料负责校验能力是否长期存在、是否能提高等级。
- 真人访谈负责修正过度判断、纠正命名偏差，并确认哪些能力可以进入最终版。
- 受阻与不确定性线索主要放到资产负债表里处理，不直接等同于能力缺陷。
- 如果一项能力既有近期沉淀证据，又有历史项目旁证，并在真人访谈中被确认，它可以进入 A 类核心资产。

## 1. 张麓的能力定位与核心结论

### 1.1 双象限定位结论

| 象限 | 张麓落位 | 含义 |
|---|---|---|
| 能力资产强度 × 商业闭环证据 | 能力资产强，商业闭环证据弱 | 能力不弱，但还没有充分证明能持续跑出商业结果 |
| 个人归因强度 × 平台依赖强度 | 个人归因较强，平台依赖不算高 | 能力大概率可以带走，但商业结果需要离开平台后重新验证 |

### 1.2 总判断

张麓不是能力不够的人，而是“能力资产强、个人归因较清楚、平台依赖不算高，但商业化落地证据还不够充分”的人。

第一步不应该急着问“能不能做产品”或“适合卖什么”，因为关键问题不在能力有无，而在：

- 哪些能力已经可以从原来的组织场景中抽离出来，成为自己的独立资产。
- 哪些能力可以进一步服务他人。
- 哪些能力进入商业化时会产生负债。

### 1.3 核心能力结论

张麓擅长复杂问题结构化和真实性校准：把模糊问题重新定义，借助已有方法论进行组合和改造，并用 AI 参与推演、追问和校准，最终把重方案压成更可判断、可验证的行动单元。

| 能力簇来源 | 支撑核心结论的部分 |
|---|---|
| 把模糊复杂问题重新定义清楚 | 先重新定义问题，再拆边界、变量和判断标准 |
| 识别真实问题和真实结果的判断力 | 判断什么是真问题、真结果、真需求、真约束 |
| 问题导向的方法论组合、改造和应用 | 把已有方法组合成当下问题可用的工具 |
| 用 AI 做推演、追问和校准 | 把 AI 当作推演对象和思维陪练，而不是答案机器 |
| 把重方案压成轻验证 | 把复杂方案压到更低风险、低成本、能先拿信号的一步 |

## 2. 能力资产地图最终版

### 能力簇 A：把模糊复杂问题重新定义清楚

定义：面对模糊、复杂、边界不清的问题，先重新定义问题本身，再拆出关键变量、边界和判断标准，把“我要做什么”改写成“我应该先判断什么”。

判定：A 类核心资产。

校准依据：

- 真人访谈中，张麓确认自己面对新问题会先泛调研和交流，但经常会质疑别人对问题的定义是否足够本质。
- 在重要工作中，会努力先把边界框出来。
- 历史 B 端平台产品训练可以作为长期旁证。

使用提醒：

- 适合作为诊断、咨询、商业切口收敛的前置能力。
- 在熟悉领域里段位较高；进入陌生领域时，需要通过专家、最佳实践和外部案例快速校准。
- 商业化时不要只交付复杂分析，要把问题重构压成用户能理解、能行动的判断结果。

### 能力簇 B：识别真实问题和真实结果的判断力

定义：能识别叙事包装、伪增长、伪需求、伪方法论和不自洽逻辑，并追问结果背后的真实归因。

判定：A 类核心资产，同时带 L 类场景型负债。

校准依据：

- 张麓会用背景、产能、交付能力、市场阶段、行业 benchmark、收入口径、真实使用情况等多个角度交叉验证。
- 这不是单纯“怀疑”，而是多变量真实性校准。
- 帮朋友校准 AI 外包定位的案例证明这项能力能服务他人，但对方一开始难以接受，也说明它有表达风险。

使用提醒：

- 只有在用户主动求诊断、愿意付费听判断、能承受被挑战时，才更容易产生价值。
- 如果用户只是想获得支持、鼓励或情绪维系，这项能力可能会让对方感到被否定。
- 商业化时要明确这是“诊断/校准服务”，并通过表达转译降低防御感。

### 能力簇 C：问题导向的方法论组合、改造和应用

定义：不是从 0 到 1 原创一套方法论，而是先锚定问题，再快速理解、选择、组合、替换和改造已有方法论，让它们变成当前问题可用、可解释、可执行的结构。

判定：A 类核心资产，但命名必须修正。

校准依据：

- 张麓明确区分了“原创方法论”和“使用、组合、改造方法论”。
- 她能很快理解复杂方法论，在使用中组合、改造、完善。
- 工作方式是先锚定问题，再选择或替换方法论工具。

使用提醒：

- 不要包装成“原创方法论专家”。
- 更适合表达为复杂方法应用者、框架组合器、问题导向的结构化顾问、把已有方法论变成可执行工具的人。
- 适合进入知识资产整理、个人方法萃取、AI 知识库框架搭建等方向。

### 能力簇 D：用 AI 做推演、追问和校准

定义：把 AI 当成推演对象、追问对象和校准工具，通过质疑、修正、重构和上下文管理逼近更准确的判断。

判定：A 类核心资产。

校准依据：

- 过去在岗位中关注 AI 原理、论文、能力边界和业务适配。
- 离岗后，重点转向人如何与 AI 协作、如何构建上下文、如何搭建工作流、如何把人机对话沉淀成知识库和 skill。
- 当前更有价值的不是 AI 技术专家身份，而是用 AI 做复杂问题推演、结构化和校准的协作能力。

使用提醒：

- 更适合探索 AI 协作工作流、AI 知识库框架、人机共创流程、AI 辅助诊断与调研。
- 不宜优先定位为 AI 技术专家、大模型原理讲师或通用 AI 工具培训师。
- 这项能力可以成为所有交付的底层工作法，也可以进入第二步商业切口假设池。

### 能力簇 E：把重方案压成轻验证

定义：把过重、过完整、过理想化的方案，压缩成低风险、低成本、能尽快拿信号的最小验证动作。

判定：A 类核心资产，但段位待提升。

校准依据：

- 张麓确认自己有“把复杂事拆成低风险、低成本、可验证小行动”的意识和动作。
- 这不是天然低压力偏好，而是经历高压大项目失败后形成的策略选择。
- 张麓也明确说自己目前“找得不够准”，容易过度设计，所以不能高估为成熟商业验证能力。

使用提醒：

- 短期不建议单独卖“商业验证方案设计”。
- 更稳妥的方式是把它作为诊断服务后半段输出：给出 1-3 个小验证动作、判断标准、成本和时间限制。
- 后续要通过真实用户、真实付费和真实复盘训练验证动作的精度。

## 3. 知识体系资产分层图最终版

一个人的知识体系可以粗分为六层：越靠上越具体、越接近执行；越靠下越抽象、越接近底层认知。

| 层级 | 张麓的知识体系资产 | 当前理解 |
|---|---|---|
| 方法论 | 问题重新定义流程、OSCAR 调研法、JTBD 变量拆解、价值萃取四问法、S3b2 诊断框架、方法论组合改造流程、AI 追问/质疑/修正流程、重方案做减法流程、低风险验证路径 | 已经出现多个可命名的方法资产；其中问题重新定义、真实性校准、AI 推演校准更成熟，轻验证动作设计还需要实战训练 |
| 思维模型 | 先定义问题再执行、问题锚定优先于方法选择、变量边界、判断维度不混淆、表面现象和真实问题区分、结果来源判断、benchmark 交叉验证、把 AI 当作思考推演对象、保留核心假设后做减法 | 解释她为什么会重构问题、追问真实结果、组合方法、压缩方案 |
| 元认知 | 观察自己的分析过程并外显成方法；识别自己何时接受、否定、修正 AI 输出；能把重大挫折、表达习惯、身份变化和转型迷茫当成可分析对象 | 能把自己的状态和能力迁移问题拆开看，而不是笼统归因为“我不行” |
| 科学真理/证据观 | 重真实结果、重口径、重归因、重可验证反馈；用背景、产能、交付能力、市场阶段、行业 benchmark 做交叉验证 | 是真实性校准能力的底层来源 |
| 人文素养 | 对转型者身份断裂、理想主义受挫、内向/高敏感用户低风险行动需求、被诊断者防御感、虚假叙事压迫感有感知 | 不适合包装成情绪价值能力，但适合帮助她筛选用户、设计低风险验证和降低服务压力 |
| 世界观/人生观/价值观 | 重真实结果、重长期主义、重独立判断、重自洽表达；不愿做低价值效率工具，不愿完全机会主义，希望短期验证能服务长期锚点 | 形成长期定力，也要求商业化路径不能只追短期热闹 |

## 4. 能力资产负债表最终版

资产不是“优点清单”，而是已经拥有、已经反复显现、未来可能被迁移和商业化的能力、知识体系、判断方式和价值偏好。

负债不是能力的反面，而是进入服务、产品和商业闭环时可能拖住她的东西。先看拖累型，再看缺失型。

### 4.1 资产侧

| 资产 | 等级 | 说明 |
|---|---|---|
| 把模糊复杂问题重新定义清楚 | A | 适合作为诊断、咨询和商业切口收敛的前置能力 |
| 识别真实问题和真实结果 | A/L | 能过滤伪需求、伪增长和包装叙事，但表达场景敏感 |
| 方法论组合、改造和应用 | A | 能把已有方法论转成当前问题可用、可解释、可执行的工具 |
| 用 AI 做推演、追问和校准 | A | 能借助 AI 追问、质疑、修正和重构判断 |
| 把重方案压成轻验证 | A | 有稳定意识和动作，但验证动作精度仍需实战提升 |
| 知识体系与价值观资产 | B | 问题定义、真实性过滤、低风险验证、长期方向感已经形成稳定性 |

### 4.2 负债侧

| 负债 | 类型 | 优先级 | 严重度 | 说明 |
|---|---|---|---|---|
| 过度分析与服务变重 | 拖累型 | P0 | 高 | 结构化能力很强，但可能让用户承接成本变高；AI 还可能扩大分析边界 |
| 判断锋利与表达压力 | 拖累型 | P0 | 中高 | 真实问题识别很快，但表达方式可能让用户防御 |
| 价值表达偏抽象 | 拖累型/缺失型 | P0 | 高 | 用户可能知道她会思考，但不知道为什么要买、买完得到什么；表达容易被评价为“人机” |
| 验证动作精度不足 | 拖累型/段位不足 | P0 | 中高 | 有轻验证意识，但最小验证动作还需要真实商业场景训练 |
| 商业闭环经验不足 | 缺失型 | P1 | 高 | 定位、获客、成交、定价、交付、复购和案例沉淀还没完整跑通 |
| 用户定位不清 | 缺失型 | P1 | 高 | 还没有锁定最适合服务、最愿意付费的人群 |
| 成交能力不足 | 缺失型 | P1 | 高 | 缺少报价、成交、拒绝处理和付费验证经验 |
| 定价经验不足 | 缺失型 | P1 | 高 | 缺少真实报价和价格测试，不清楚诊断、咨询、项目和陪跑的价格边界 |
| 标准化交付不足 | 缺失型 | P1 | 中高 | 方法很多，但还需要压成用户可理解、可完成、可验收的最小服务单元 |
| 结果定义不足 | 缺失型 | P1 | 中高 | 她很重视真实结果，但自己的服务如何证明有效还不清楚 |
| 外部案例和结果证明不足 | 缺失型 | P1 | 高 | 强证据主要来自自我问题和 AI 对话，还缺服务他人的外部证据 |
| 行动闭环不足 | 缺失型 | P1 | 中 | 有轻验证意识，但实际执行、反馈和复盘证据偏少 |
| 慢思考环境依赖 | 拖累型 | P2 | 中 | 更适合文本、AI 对话、异步沉淀和低压力共创 |
| 人群适配变窄 | 拖累型 | P2 | 中 | 真实、长期主义、独立判断的偏好可能成为差异化，也可能天然筛掉一部分用户 |

### 4.3 负债处理优先级

| 优先级 | 先处理什么 | 原因 |
|---|---|---|
| P0 | 过度分析、判断锋利、价值表达偏抽象 | 必须通过服务边界、时间盒、交付格式和表达转译先约束住 |
| P0 | 低风险验证动作精度不足 | 避免“轻验证被设计重” |
| P1 | 用户定位、获客、成交、定价、交付闭环没有跑过 | 这是缺失型负债，从白纸验证即可，不需要先纠偏旧路径 |
| P1 | 缺少外部案例和结果证明 | 决定这些能力是否真的能服务他人，而不只是自我分析强 |
| P2 | 慢思考环境依赖、人群适配变窄 | 这些不是要被消灭，而是用来筛选适配商业模式和适配用户 |

## 5. 第二步候选能力池与选择理由

能力只是第二步的输入，不等于商业方向答案。真正选择商业切口前，还需要补资源、时间、渠道、偏好、边界和风险承受力。

### 5.1 建议进入第二步的能力

| 能力 | 适合探索的商业切口 | 必须带着处理的负债 |
|---|---|---|
| 把模糊复杂问题重新定义清楚 | 转型期问题诊断、产品方向诊断、个人能力商业化诊断 | 过度分析、标准化交付不足 |
| 识别真实问题和真实结果 | 项目/产品/个人定位的真实性校准、商业假设体检 | 判断锋利、表达过硬、外部案例不足；必须筛选能承受诊断的用户 |
| 问题导向的方法论组合、改造和应用 | 一对一方法萃取、经验方法论整理、个人知识资产整理、AI 知识库框架搭建 | 价值表达偏抽象、交付边界不清 |
| 用 AI 做推演、追问和校准 | AI 协作工作流设计、AI 知识库框架、人机共创流程、AI 辅助诊断与调研 | 避免做成泛 AI 工具课；要绑定具体问题和交付结果 |
| 把重方案压成轻验证 | 作为诊断后的后半段输出：1-3 个小验证动作、判断标准、成本和时间限制 | 暂不单独作为主卖点；必须通过真实复盘训练精度 |

### 5.2 第二步优先探索方向

| 优先级 | 方向 | 为什么 |
|---|---|---|
| P0 | 复杂问题结构化与真实性校准诊断 | 最能承接 A 类能力：问题重新定义、真实问题识别、方法组合和 AI 推演校准 |
| P0 | 转型期个人/专家的能力商业化诊断 | 与她自身近期问题高度贴近，也能用能力资产负债表帮助用户减少自嗨和自我贬低 |
| P1 | 能力地图 + AI 知识库框架搭建 | 访谈中张麓对这个方向反应很强，值得进入假设池；但还要验证用户是否愿意付费 |
| P1 | 项目/产品方向的真实性校准 | 能发挥识别伪需求和伪结果的判断力，但需要注意表达方式和用户承受度 |
| 暂缓 | AI 工具课、泛产品原型服务、To Bank 独立咨询 | 要么同质化，要么过重，要么强依赖历史平台经验，不适合作为当前第一切口 |

## 6. 第二步之前仍需补充的信息

第一步已经解决的是：能力资产是什么、负债是什么、哪些能力值得进入商业切口选择。

第一步还不能直接解决的是：最终应该选哪个商业方向、卖什么服务、服务谁、定什么价、从哪里获客。因为这些不能只从能力推出，还必须结合现实边界。

进入第二步前，建议补充：

- 当前可投入的时间、精力和现金流压力。
- 当前可触达的人脉、社群、渠道和可信背书。
- 最不想做的服务形式、最不能承受的压力类型。
- 更愿意服务个人、团队、企业，还是特定圈层里的转型者。
- 对公开内容、私域成交、一对一诊断、陪跑、课程、项目制交付的偏好。
- 愿意用多低成本、多短周期先验证第一单。

下一步的正确动作不是直接定方向，而是基于这些边界，生成 3-7 个商业切口假设，再做轻量调研和最小付费验证计划。
